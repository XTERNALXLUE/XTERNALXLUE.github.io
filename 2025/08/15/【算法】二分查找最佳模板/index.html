二分查找最佳模板

先看代码（数组a下标从0开始）

这是查找有序数组值为x的右边界

即最后一个 <= x 的元素下标

找的是右边界，但最后一个满足条件时更新的是 l = mid，所以返回 l

再给出最后查找x的左边界

即第一个 >= x的元素下标

找的是左边界，但最后一个满足条件时更新的是 r = mid，所以返回 r

为什么说这是最佳模板？

更新的时候不用考虑+1，-1问题

这个模板唯一要考虑的就是：

更新时满足条件的元素，是归属到“左边”还是“右边”

拿第一段代码举例

要找的是最后一个 <= x 的元素下标

也就是值为x的右边界（不存在为x的值时，就是 < x 的右边界）

比如 1，2，3，3，4，5…
x = 3时

如果当前mid指向第一个3，那他一定归属与这个问题的“左边”

直接更新 l = mid 即可

如果指向第二个3，也是一样他一定归属于“左边”
也直接更新 l = mid

对应代码中 if(a[mid] <= x) l = mid

其他情况自然就是归属“右边”，更新 r = mid 即可

（第二段代码找左边界同理）

其他的都是固定的，只需要记住，不需要考虑

1.左右边界初始值

l 和 r 初始值为“这段连续存储的数组之外的第一位没有元素的下标”

一般下标从0开始，即 0 ~ n - 1 有元素

那 l 和 r 分别就是没有元素的 -1 和 n

如果下标从1开始，那就是 1 ~ n有元素

那 l 和 r 分别就是没有元素的 0 和 n + 1

2.while循环条件

考虑极端情况：数组中只有一个元素2，（下标从0开始）

那根据第1点提到的

l = -1, r = 1;

因为至少要返回左边界和右边界的一个

此时直接返回肯定不行

必须进入1次while循环

那么此时满足 l 和 r 关系的真语句就是l + 1 < r

只要理解了唯一要考虑的事情：归属“左边”还是“右边”

自己再举实际例子去看，就很明了了
